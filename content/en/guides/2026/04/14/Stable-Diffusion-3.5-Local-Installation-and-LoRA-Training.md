## 🎯 Overview
This guide details the advanced workflow necessary to install the large-scale Stable Diffusion 3.5 model locally and execute a comprehensive LoRA fine-tuning process. The objective is to establish a reproducible, high-performance environment using virtual environments and dedicated training pipelines.

## 🚀 Phase 1. Infrastructure Setup: Core Environment Build
* **Step 1-1. Create and Activate Virtual Environment**
  * *Purpose:* Isolates project dependencies from the global Python installation, preventing version conflicts.
  * `python3 -m venv sd3_venv`
  * `source sd3_venv/bin/activate`
  * **Expected Result:** The command prompt prefix changes to `(sd3_venv)`, indicating the virtual environment is active.

* **Step 1-2. Install Dependencies (PyTorch/ComfyUI)**
  * *Purpose:* Installs the necessary deep learning framework and the primary UI tool (ComfyUI/Forge). Note: CUDA compatibility is assumed.
  * `pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121`
  * `git clone https://github.com/comfyanonymous/ComfyUI`
  * `cd ComfyUI`
  * `pip install -r requirements.txt`
  * **Expected Result:** PyTorch and all specified dependencies are installed successfully. The `ComfyUI` directory contains all necessary executables and custom nodes.

* **Step 1-3. Download Model Weights**
  * *Purpose:* Acquiring the massive Stable Diffusion 3.5 base model checkpoint.
  * `wget https://huggingface.co/path/to/sd3.5/resolve/main/sd3.5_v1.zip`
  * `unzip sd3.5_v1.zip -d ./models/checkpoints`
  * **Expected Result:** The `models/checkpoints` directory within the ComfyUI structure contains the primary `sd3.5` model file (e.g., `sd3.5.safetensors`).

## ⚙️ Phase 2. LoRA Training Setup and Execution
* **Step 2-1. Prepare Training Environment (Kohya)**
  * *Purpose:* LoRA training often requires dedicated tools like Kohya's scripts, which handle dataset preparation and training loops efficiently.
  * `git clone https://github.com/kohya-ss/sd-scripts`
  * `cd sd-scripts`
  * `pip install -r requirements.txt`
  * **Expected Result:** The `sd-scripts` folder is populated with all necessary training utilities, including the `train_lora.py` script.

* **Step 2-2. Dataset Preparation and Alignment**
  * *Purpose:* Organizing source images and creating captions. All source images must be resized and aligned to a consistent resolution (e.g., 512x512 or 1024x1024).
  * *Action:* Manually create a structured folder: `./training_data/my_subject/` containing `image_001.jpg`, `image_002.jpg`, etc., and a corresponding `caption_001.txt`.
  * **Expected Result:** A clean, organized dataset directory ready for batch processing by the training script.

* **Step 2-3. Execute LoRA Training Script**
  * *Purpose:* Running the command-line interface to fine-tune the base model weights using the prepared dataset.
  * `python train_lora.py --pretrained_model_name_or_path="[Path to SD 3.5]" --output_dir="[Path to output]" --train_data_folder="training_data/my_subject" --resolution=1024 --train_batch_size=1 --max_train_steps=1000 --learning_rate=1e-5 --network_dim=64`
  * **Expected Result:** The script initiates the training process, outputting progress logs (e.g., loss values) and generating optimized LoRA checkpoints (`.safetensors`) in the specified output directory.

## 🚀 Phase 3. Inference and Testing
* **Step 3-1. Integrate LoRA into ComfyUI**
  * *Purpose:* Loading the newly trained LoRA weights into the ComfyUI workflow for image generation.
  * *Action:* Place the resulting LoRA file (e.g., `my_subject_v1.safetensors`) into the ComfyUI `models/loras` directory.
  * **Expected Result:** The LoRA file is recognized by ComfyUI nodes, allowing the user to select it in the workflow UI.

* **Step 3-2. Generate Final Image**
  * *Purpose:* Running a final generation pass using the base model, the LoRA weights, and a refined prompt.
  * *Action:* In ComfyUI, load the SD 3.5 checkpoint, load the specific LoRA, and execute the generation graph with a detailed prompt (e.g., "A cinematic portrait of [Subject LoRA], highly detailed, volumetric lighting").
  * **Expected Result:** A high-fidelity, custom-styled image generated by the base model, specifically modified by the fine-tuned LoRA weights.
