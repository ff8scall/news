# 🏛️ 레고-시아 V2.6: Strategic Automation Hardening 체크리스트

이 문서는 사용자 피드백을 반영하여 News + Guide 시스템의 안정성과 품질을 극대화하기 위한 최종 보완 계획입니다.

---

## 🛠️ 섹션 1: 엔진 안정화 (Hardening) - [READY]
- [ ] **Safe JSON Parser**: `ai_news_editor.py` 및 `ai_guide_editor.py`에 ```json 블록 제거 및 정밀 추출 로직 주입.
- [ ] **Token Guard**: 가이드 생성 시 뉴스 본문 참조 길이를 최대 2,500자로 제한하여 토큰 폭주 방지.
- [ ] **Category-Slug Sync**: 구형 언더바(_) 슬러그를 하이픈(-) 기반 최신 슬러그 및 4대 클러스터 명칭으로 전면 교체.
- [ ] **Error Handling**: AI 호출 실패 시 `None` 반환 및 메인 루프에서의 예외 처리 강화.

## 📗 섹션 2: 가이드 엔진 실전 배치 (Guide Deployment) - [READY]
- [ ] **Guide Editor 전면 가동**: 비어있는 `_call_ai` 메서드를 `AIWriter` 인터페이스와 연결하여 실제 집필 시작.
- [ ] **Guide Type Classification**: `match_counts`를 분석하여 가이드 유형(tutorial, tuning, troubleshooting 등) 자동 분류.
- [ ] **Daily Publish Limit**: 하루 최대 가이드 발행량을 3~4건으로 제한하는 카운터 로직 추가.
- [ ] **create_guide_post 구현**: `/guides/[type]/[slug]` 경로에 최적화된 마크다운 생성기 구축.

## 🔗 섹션 3: 데이터 핸드셰이크 (Main Loop) - [READY]
- [ ] **Full-Link Integration**: `news_main.py`에서 `GUIDE_READY` 감지 시 즉시 `write_guide` -> `create_guide_post` 호출.
- [ ] **Sync-Slug Persistence**: 뉴스와 가이드가 동일한 `sync_slug`를 사용하여 상호 참조(SEO) 가능하게 정렬.
- [ ] **Status Reporting**: 텔레그램 리포트에 가이드 발행 성공 여부 및 개수 포함.
