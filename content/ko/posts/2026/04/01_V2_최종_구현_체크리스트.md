# 🏛️ 레고-시아 뉴스 엔진 V2 최종 구현 체크리스트

이 문서는 V2 "Strategic Intelligence" 엔진의 설계부터 운영까지 모든 단계를 검증하기 위한 최종 체크리스트입니다.

---

## 1단계: 수집 및 필터링 엔진 (Harvester & Scorer) - [DONE]
가장 먼저 데이터의 '질'을 결정하는 관문입니다.

- [x] **Tier 시스템 구축**: RSS/API 소스별로 Tier 1(1.0), Tier 2(0.8), Tier 3(0.6) 가중치 테이블이 매핑되었는가? (완료: news_harvester.py)
- [x] **스코어링 로직 구현**: Editorial Score 공식이 정확히 계산되는가? (완료: news_main.py)
- [x] **문항 추출(Extraction) 엔진**: 각 기사에서 마케팅 문구를 제외하고 숫자/제품명/가격이 포함된 핵심 5문장을 뽑아내는 NLP/Regex 로직이 작동하는가? (완료: extract_core_sentences)
- [x] **토큰 절감 확인**: 원문 전체를 보낼 때 대비 토큰 사용량이 50~60% 이상 감소했는가?

## 2단계: 지능형 그룹화 레이어 (Intelligence Layer) - [DONE]
중복을 막고 '사건'으로 묶는 시스템의 심장부입니다.

- [x] **유사도 알고리즘**: Jaccard Similarity 기준값 0.80~0.85가 그룹화에 적절히 적용되는가? (기준 0.80 적용)
- [x] **분열 방지(Fragmentation Guard)**: 3시간 이내의 유사 사건(0.75 이상)이 감지되면 새 리포트를 만들지 않고 '병합' 프로세스로 넘어가는가?
- [x] **메가 이벤트 감지**: 6개 이상의 소스가 묶였을 때 `Mega_Event = True` 플래그가 생성되고, 리포트 분량(+30%) 가중치가 부여되는가?

## 3단계: AI 에디터 & 생성 프로세스 (The Brain) - [DONE]
수석 편집자의 페르소나가 투영되는 단계입니다.

- [x] **시스템 프롬프트 주입**: 제공된 15년 경력 수석 에디터 페르소나가 System Instruction에 완벽히 이식되었는가? (완료: NewsEditor)
- [x] **이중 언어 추론(Internal Reasoning)**: AI가 내부적으로 영문 소스를 비교 분석한 뒤 한국어로 출력하는 프로세스가 안정적인가? (Stage 1 English -> Stage 2 Korean)
- [x] **7단계 구조 준수**: 제목부터 ## What This Means까지 누락된 섹션 없이 출력되는가?
- [x] **톤앤매너 검수**: 번역투가 아닌, 한국 IT 전문가의 드라이하고 날카로운 브리핑 톤이 유지되는가?

## 4단계: 생존 및 복구 시스템 (Survivability) - [DONE]
운영자가 자고 있을 때 시스템을 지켜주는 방어기제입니다.

- [x] **시간 제어(Time Control)**: 2시간 사이클당 최대 발행량이 제한되며, 초과분은 Queue로 정상 이동하는가? (완료: 메인 루프 예산 제어)
- [x] **실패 복구(Fail-Safe)**: LLM API 타임아웃 발생 시, 즉시 3줄 요약 형태의 Short Brief가 생성되는가? (완료: Fail-Safe Fallback 구현)
- [x] **이벤트 캐시(Update)**: 기존 사건에 새 소스 추가 시 리포트 전체 재생성이 아닌 특정 세션만 '업데이트'되는가? (완료: Delta Update 로직)
- [x] **데드 필터**: 48시간 이상 업데이트 없는 사건이 자동으로 아카이브 폴더로 이동하는가? (완료: Archive Cron Job)

## 5단계: 비주얼 및 AI 미디어 (Visual Engine) - [DONE]
사이트의 프리미엄 디자인 완성도를 결정합니다.

- [x] **폴백(Fallback) 완성도**: 카테고리별 마스터 폴백 이미지가 100% 준비되어 있는가? (완료: fallback-default 생성)
- [x] **이미지 우선순위**: 원본 -> 마스터 폴백 -> AI 생성 순위가 코드에 정밀하게 반영되었는가?

## 6단계: 검색 엔진 지배력 (SEO & Distribution) - [DONE]
발행된 기사가 실제 트래픽으로 이어지게 하는 단계입니다.

- [x] **IndexNow 자동화**: 기사 발행 즉시 주요 검색 엔진(Bing, Naver 등)에 URL을 실시간 통보하는가? (완료: news_main.py)
- [x] **내부 링크 구조**: 관련 있는 과거 기사들을 `Related Posts` 섹션에 지능적으로 연결하는가? (완료: Vector DB 기반 연관성 매칭)
- [x] **다국어 매칭**: KO/EN 기사 간의 상관관계가 메타 데이터에 기록되는가? (완료: Metadata Mapping)

## 7단계: V2.4 하이브리드 지능 플랫폼 (Hybrid Intelligence) - [DONE]
뉴스에서 가이드와 비교 자산을 자동으로 추출하는 최신 레이어입니다.

- [x] **Guide Candidate Detection**: `Rule G1~G3`에 기반한 가이드 후보 감지 로직이 가동되는가? (완료: evaluate_guide_potential)
- [x] **Compare Trigger Logic**: `Rule C1~C3`에 기반한 비교 트리거 로직이 가동되는가? (완료: evaluate_compare_potential)
- [x] **Slug Sync System**: 영문 제목 기반 단일 슬러그 체계가 KO/EN 포스팅에 동기화되는가? (완료: sync_slug)
- [x] **운영 예산 배분**: 4:3:2:1 (Intelligence : Physical : Strategy : Digital) 비율이 유지되는가?

---

💡 **전문가의 최종 조언**: 
모든 시스템이 V2.4 "Hybrid Intelligence" 단계까지 무결하게 업그레이드되었습니다. 이제 시스템은 수동 개입 없이도 뉴스와 정보를 넘어 교육(Guides)과 의사결정 지원(Compare)을 하는 고도의 테크 미디어로 기능합니다.
