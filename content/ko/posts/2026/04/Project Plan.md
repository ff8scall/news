# news.lego-sia.com 프로젝트 구축 상세 분석 및 실행 계획서 (Expert Revised V1.1)

본 문서는 Hugo 정적 사이트 생성기를 활용하여 캠핑을 시작으로 다양한 라이프스타일 주제로 확장 가능한 프리미엄 뉴스 플랫폼 `news.lego-sia.com`의 구축 계획을 상세히 담고 있습니다. 
**전문가의 시각에서 잠재적 리스크를 보완하고, 웹 접근성 제고, 확장성과 SEO 완성도를 극대화**하기 위한 디테일을 보충하였습니다.

## 1. 프로젝트 개요 및 비전
### 1.1 플랫폼 아이덴티티
* **서비스 명**: news.lego-sia.com
* **핵심 가치**: 전문성 있는 정보(캠핑, IT, 요리)와 세련된 디자인의 조화
* **사용자 경험(UX) 목표**: "읽기 즐거운, 찾기 쉬운, 보기 아름다운" 정적 웹사이트
* **타겟 오디언스(전문가 추가)**: 깊이 있는 정보를 원하는 20~40대 직장인 및 취미 생활자
* **핵심 성공 지표/KPI(전문가 추가)**: 초기 3개월 내 유기적 검색(Organic) 유입 비중 70% 달성, 페이지 로드 속도(LCP) 1.5초 이내 유지.

### 1.2 기술 스택 선정 이유 (보완점 포함)
| 구성 요소 | 기술 | 선정 이유 및 전문가 보완 포인트 |
|---|---|---|
| **Generator** | Hugo | 빌드 속도가 압도적이며, 추후 수천 개의 포스트가 쌓여도 확장에 방어 가능함. |
| **Content** | Markdown / Frontmatter | 메타데이터(Frontmatter)를 엄격하게 구조화하여 카테고리/태그 연동을 체계화 설계. |
| **Styling** | Vanilla CSS / SCSS | 외부 라이브러리 의존도를 낮춰 경량화하되, `assets` 폴더 내 SCSS 또는 CSS 변수(Token)를 구축해 체계적인 테마 시스템 구현. |
| **Interactivity** | Vanilla JS / Pagefind | 무거운 Client-side JS 대신 정적 검색 라이브러리인 `Pagefind`를 활용하여 퍼포먼스 저하를 방지. |
| **Deployment** | Netlify | 배포 자동화 및 PR Preview를 통해 생성되는 마크다운 포스트를 실제 환경에서 미리 검수 가능. |

## 2. 아키텍처 및 디렉토리 구조 (Advanced)
Hugo의 유연한 Content Organization 기능을 활용하며, 향후 유지보수를 위해 구조를 고도화합니다.

```text
/
├── docs/                # 💡 프로젝트 기획/설계, 산출물 문서 전용 보관소 (현재 문서 위치)
├── archetypes/          # 콘텐츠 템플릿 (카테고리별 Frontmatter 필드 분리)
├── content/             # 실제 마크다운 파일 보관
│   ├── camping/         
│   ├── travel/          
│   ├── cooking/         
│   └── it/              
├── data/                # 네비게이션 메뉴, 카테고리 설정, 저자(Author) 프로필 전역 데이터 설정
├── layouts/             # HTML 템플릿 구조 (Custom Design)
│   ├── partials/        # footer, header, SEO 메타태그 등 재사용 컴포넌트 모듈
│   └── shortcodes/      # 💡 Markdown 내에서 호출 가능한 커스텀 확장 뷰 (알림 상자, 유튜브 임베디드 등)
├── static/              # favicon, robots.txt, 구글 인증 파일 등 정적 파일
├── assets/              
│   ├── css/             # 💡 index.css 및 디자인 시스템 변수 
│   └── js/              # 핵심 기능 JS 모듈화
└── hugo.toml            # 메인 환경설정
```

## 3. SEO 및 URL 전략 (리스크 및 기술 보강)
### 3.1 Advanced SEO 최적화 전문가 권장 사항
* **JSON-LD 구조화 데이터 추가**: Article, Breadcrumb 스키마뿐만 아니라 추후 **검색엔진용 FAQ 스키마, Author 프로필 스키마**를 생성하도록 템플릿화 적용.
* **Canonical URL 명시**: 중복 콘텐츠로 인한 크롤링 페널티 방지를 위해 모든 페이지 헤더에 `ref="canonical"` 명시적 적용 필수.
* **이미지 WebP 자동화**: Hugo Image Processing 기능을 활용해 첨부된 원본 이미지를 `WebP`로 자동 압축 적용하여 페이지 렌더링 성능 극대화.

### 3.2 한글 URL 사용 및 공유 리스크 대응
* **포맷 설정**: `/:section/:slug/`
* **고려사항**: 한글 URL(`removePathAccents = false`)은 직관적이나 카카오톡이나 SNS 공유 시 인코딩이 풀려(%EC%A1%B0...) 미관상 좋지 않고 길어지는 현상이 있습니다.
  * **보완 전략**: 기본적으로 한글 URL을 유지하여 높은 직관성을 확보하되, `og:url` 등 메타 영역에서의 처리나 필요시 Frontmatter에 라틴(영문) `slug` 값을 별도로 작성하여 안정성을 헷지하는 하이브리드 전략을 취합니다.

## 4. 디자인 시스템 (Premium UX/A11Y)
단순한 뉴스를 넘어, 접근성과 디테일이 살아있는 프리미엄 느낌의 UI를 구현합니다.

### 4.1 타이포그래피, 테마 및 웹 접근성
* **Primary Font**: Pretendard (캐싱용 경량화 woff2 적용 및 preload 렌더링 최적화).
* **유연한 Color Palette (CSS Variables)**:
  * Dark Mode (Default): `var(--bg-dark): #0F172A`, `var(--accent-primary): #38BDF8`
  * 기초 테마 토큰을 설계하여 추후 Light Mode 전환 토글 버튼 추가 확장성을 확보.
* **접근성(A11Y) 향상**: 프리미엄 한 디자인은 본문 가독성에서 나옵니다. 배경과 텍스트의 명도 대비(Contrast Ratio)의 W3C 웹 접근성 표준을 준수.

### 4.2 핵심 인터랙션 및 컴포넌트 세분화
* **Sticky & Glassmorphism Header**: 스크롤 시 상단에 고정되며 컨텐츠가 뒤로 비치는 블러 효과 적용.
* **Dynamic 동적 목차(TOC)**: 긴 글 읽기의 길잡이 역할. `IntersectionObserver` API를 활용하여 화면 스크롤 시 현재 읽고 있는 파트에 하이라이트 효과 적용.
* **추천 글 및 카테고리 필터링**: 유사한 `Tag` 기반의 콘텐츠 자동 추천 클러스터 노출.

## 5. 초기 콘텐츠 로드맵 및 분류 체계 (Taxonomy) 전략
콘텐츠의 무한한 확장을 위해 Hugo의 Taxonomy(분류 체계) 정교화 셋팅을 제안합니다.
**분류 규칙 설계**: 
- `Category`: 대분류 폴더 형식 (`Camping`, `IT`, `Cooking`)
- `Tag`: 상세 분류 해시태그 형식 (`#초보가이드`, `#예약꿀팁`)

**초기 캠핑 중심 9선 분류 예시**:
1. 초보 캠핑장 예약 팁 2026 최신 가이드 (Tag: `#초보자가이드`)
2. 봄 테마별 추천 캠핑장 TOP 10 (Tag: `#봄`, `#추천`)
3. 여름 테마별 추천 캠핑장 TOP 10 (Tag: `#여름`)
4. 가을 테마별 추천 캠핑장 TOP 10 (Tag: `#가을`)
5. 겨울 테마별 추천 캠핑장 TOP 10 (Tag: `#겨울`, `#추천`)
6. 캠핑장 예약 취소 시 환불 규정 총정리 (Tag: `#규정자료`)
7. 인기 캠핑장 오픈 시간 알림 설정 기법 (IT 연계) (Tag: `#예약꿀팁`, `#IT융합`)
8. 평일 vs 주말 예약 성공률 비교 분석 인포그래픽 데이터 (Tag: `#데이터분석`)
9. 안전/체크리스트 계절별 필수 준비물 (Tag: `#안전수칙`)
