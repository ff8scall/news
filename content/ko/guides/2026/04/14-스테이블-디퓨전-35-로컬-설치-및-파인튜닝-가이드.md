### 🏁 INTRO: 이 가이드가 중요한 이유

안녕하세요, 수강생 여러분.

최신 AI 모델을 직접 운영하는 것은 단순한 사용을 넘어선 기술적 숙련도를 의미합니다.

SD 3.5는 이전 모델 대비 압도적인 이해력과 디테일을 자랑합니다.

하지만 클라우드 기반 서비스는 지연 시간(Latency)이나 비용 효율성 면에서 한계를 가집니다.

이 가이드는 여러분이 최고의 성능을 내는 로컬 환경을 구축하도록 돕습니다.

복잡하게 느껴질 수 있지만, 제가 100% 성공할 수 있는 단계를 안내해 드릴게요.

> 🛠️ PREREQUISITES: 준비물 확인

성공적인 시작을 위해 다음 도구와 환경이 반드시 갖춰져야 합니다.

*   **하드웨어:** NVIDIA GPU (최소 VRAM 12GB 이상 권장)
*   **운영체제:** Windows 11 또는 Ubuntu 22.04 이상
*   **소프트웨어:** Python 3.10.x (가상 환경 권장), Git, CUDA Toolkit 12.1


> 📝 STEP-BY-STEP: 10단계 성공 로드맵

다음 순서대로 따라오시면 오류 없이 시스템을 구축할 수 있습니다.

각 단계를 완료할 때마다 결과를 확인해 주세요.

**1.

가상 환경 설정 및 의존성 설치**

*   **Action:** `conda`를 사용하여 격리된 가상 환경을 생성하고 필수 라이브러리를 설치합니다.

*   **Result:** 시스템 충돌 없이 독립적으로 SD 3.5 환경을 구축할 준비가 완료됩니다.

**2.

PyTorch 및 CUDA 연동 확인**

*   **Action:** 설치된 PyTorch가 로컬 GPU와 정확하게 연동되는지 검증합니다.

*   **Result:** GPU 자원을 100% 활용할 수 있는 최적의 환경이 확보됩니다.

**3.

SD 3.5 핵심 모델 다운로드**

*   **Action:** 공식 저장소에서 최신 버전의 SD 3.5 체크포인트와 토크나이저를 다운로드합니다.

*   **Result:** 고품질 이미지 생성을 위한 핵심 모델 파일들이 준비됩니다.

**4.

API v4 엔드포인트 설정**

*   **Action:** 모델을 외부 애플리케이션에서 호출할 수 있도록 API 게이트웨이를 설정합니다.

(아래 YAML 참조)
*   **Result:** 안정적이고 빠르며, 확장성이 높은 API v4 통신 채널이 구축됩니다.

**5.

기본 이미지 생성 테스트**

*   **Action:** 간단한 프롬프트로 기본 이미지 생성을 실행하여 시스템의 안정성을 테스트합니다.

*   **Result:** 지연 시간(Latency)과 해상도가 예상 범위 내에 있는지 확인합니다.

**6.

배치 처리 최적화 (Batch Size 설정)**

*   **Action:** 한 번에 여러 이미지를 처리할 수 있도록 배치 크기(`batch_size`)를 조정합니다.

*   **Result:** 처리 속도가 획기적으로 향상되어 대량 작업에 유리합니다.

**7.

파인튜닝 데이터셋 준비**

*   **Action:** 커스터마이징할 스타일이나 캐릭터의 이미지를 수집하고, 일관된 포맷으로 폴더를 구성합니다.

*   **Result:** 고품질의 학습 데이터셋이 준비되어 모델 학습을 위한 기반이 마련됩니다.

**8.

LoRA 학습 스크립트 실행**

*   **Action:** 준비된 데이터셋과 학습 파라미터를 사용하여 LoRA(Low-Rank Adaptation) 모델을 학습시킵니다.

*   **Result:** 특정 스타일이나 캐릭터에 특화된 고유한 LoRA 가중치 파일이 생성됩니다.

**9.

모델 병합 및 검증**

*   **Action:** 기본 모델과 새로 학습된 LoRA를 병합하여 최종 커스터마이징 모델을 만듭니다.

*   **Result:** 최적화된 최종 모델을 통해 원하는 결과를 안정적으로 얻을 수 있습니다.

**10.

최종 API 통합 및 배포**

*   **Action:** 완성된 모델을 API v4 환경에 재배포하고, 외부 서비스와 연동 테스트를 진행합니다.

*   **Result:** 상업적 활용이 가능한, 완벽하게 최적화된 로컬 AI 시스템이 완성됩니다.

> 💻 CODE/CONFIG: API 설정 예시 (config.yaml)

API의 효율성과 리소스를 관리하는 설정 파일입니다.

이 내용을 반드시 확인해주세요.

```yaml
api_v4:
  endpoint: "http://localhost:8000/api/v4/generate"
  max_concurrent_jobs: 8
  gpu_memory_limit_gb: 10.0
  batch_size_optimized: 4
  quantization_level: 'fp16'
```


> 💡 EXPERT TIPS: 성능 극대화 팁

*   **양자화(Quantization) 활용:** 모델 로딩 시 `fp16` 또는 `int8` 같은 저정밀도 양자화를 사용하세요.

이는 VRAM 사용량을 획기적으로 줄여줍니다.

*   **스케줄링:** 학습 시 GPU 사용률이 90% 이상을 유지하도록 배치 크기와 스텝 수를 조절하는 것이 가장 중요합니다.

*   **커널 최적화:** PyTorch에서 제공하는 커널 최적화 툴을 활용하여 메모리 액세스 병목 현상을 미리 점검하세요.

> ⚠️ TROUBLESHOOTING: 흔한 오류 해결법

**1.

[CUDA Out of Memory] 오류 발생 시:**

*   **원인:** GPU VRAM이 부족하여 모델을 로드할 공간이 없습니다.

*   **해결:** `batch_size`를 1로 줄이거나, 설정 파일에서 `quantization_level`을 반드시 `fp16`으로 설정하여 메모리 사용량을 절반으로 줄여주세요.

**2.

API 호출 시 'Connection Refused' 오류:**

*   **원인:** API 서버가 제대로 실행되지 않았거나 방화벽이 포트를 막고 있습니다.

*   **해결:** 서버 프로세스가 백그라운드에서 정상 작동하는지 확인하고, `config.yaml`에 명시된 포트(예: 8000)가 열려 있는지 점검하세요.

**3.

파인튜닝 결과가 만족스럽지 않을 때:**

*   **원인:** 데이터셋의 다양성이 부족하거나, 데이터 전처리 과정에서 노이즈가 포함되었을 수 있습니다.

*   **해결:** 데이터셋을 캡션(Captioning) 단계에서 수작업으로 검토하고, 다양한 각도와 조명 조건의 이미지를 추가하여 재학습을 시도하세요.
